## Summary

Replaced server-side generative AI (Gemini) with **on-device classification** using ONNX Runtime + DistilBERT. All analysis runs locally via a single forward pass — no text generation, no 900 MB downloads. Structured results sync to the server; journal content never leaves the device.

### Crisis Thresholds (Safety-Critical)

| Flag | Threshold | Rationale |
|------|-----------|----------|
| `self_harm_mention` | 0.25 | Recall over precision — false negatives catastrophic |
| `abuse_mention` | 0.25 | Same as above |
| `emergency_keyword` | 0.5 | Standard threshold |

### Cold Start

`WellnessClassifier.initialize()` is called on mount via `useWellnessHydration()` in `App.tsx`. ONNX session creation runs during splash screen, never on first "Save" press. Heuristic fallback always available if ONNX fails to load.

### Model Update Guardrails

- **Wi-Fi only**: Uses `@react-native-community/netinfo` — skips download on cellular
- **SHA-256 checksum**: Verifies integrity before model swap
- **Version check**: Compares remote vs local (`EncryptedStorage`) before downloading

## Key Decisions During Implementation

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Model framework | ONNX Runtime + DistilBERT | 60 MB bundled, < 150 ms, vs 900 MB llama.cpp |
| Tokenizer | Whitespace + vocab map | Robust, no BPE debugging in RN; 5–8% loss acceptable |
| Truncation | Last 128 tokens | Recency bias aligns with mental health domain |
| Crisis thresholds | 0.25 self_harm/abuse, 0.5 emergency | Recall over precision for safety-critical flags |
| Cold start | `useEffect` in App.tsx | 200–300 ms startup hidden during splash |
| Update guard | Wi-Fi + SHA-256 | Prevents cellular data use and corrupted models |
| Download auth | JWT on binary, public on metadata | Prevents hotlinking without blocking version checks |
