## Features That Actually Work Fully Offline

- ✅ Cycle prediction — on-device ML model cached in EncryptedStorage with bundled fallback
- ✅ Wellness sentiment classification — ONNX model runs locally (once downloaded)
- ✅ Journal drafts — auto-saved every 30s to EncryptedStorage
- ✅ Auth token/user caching — persisted in EncryptedStorage for app restart
- ✅ Native SMS SOS fallback — bypasses API entirely
- ✅ Mock data features (36 screens) — show data but it's fake

---

## Features That SHOULD Work Offline But Don't (Critical Gaps)

- ❌ Writing anything (mood, journal, period, SOS, contacts) — fails with error toast
- ❌ Reading previously fetched data after restart — no persistQueryClient
- ❌ Period log entry — not even submitted online, data lost on navigation
- ❌ SOS trigger offline — safetySyncQueue.ts exists but unwired, user gets navigated back on failure
- ❌ ConnectivityBanner says "changes saved locally" — they are not saved for any write feature
